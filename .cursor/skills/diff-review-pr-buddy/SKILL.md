# Diff Review / PR Buddy

**Use when:** user pastes a diff or asks for review

## Steps

1. Summarize intent
2. Flag correctness risks (edge cases, UB, concurrency)
3. Flag style/consistency issues
4. Suggest 1–3 concrete improvements
5. Provide test plan
